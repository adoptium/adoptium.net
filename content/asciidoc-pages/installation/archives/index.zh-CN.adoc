== Linux

. 确保您已将最新的 link:/download[Linux 二进制文件] 下载到一个不会移动或删除的目录中，并使用终端 cd 进入该目录。
. 可选：使用这些校验和说明来确保您的二进制文件的真实性：
+
[source, bash]
----
# 命令必须在下载的二进制文件所在的同一目录中
wget -O- -q -T 1 -t 1 <url_to_checksum_file> | sha256sum -c
----
+
. 解压 `.tar.gz`。您可以使用以下命令：
+
[source, bash]
----
tar xzf <openjdk_binary>.tar.gz
----
+
. 将此版本的 Java 添加到您的 `PATH`
+
[source, bash]
----
export PATH=$PWD/<extracted_directory>/bin:$PATH
----
+
. 检查 Java 是否已正确安装：
+
[source, bash]
----
java -version
----

== Windows

. 确保您已将最新的 link:/download[Windows 二进制文件] 下载到一个不会移动或删除的目录中，并使用命令提示符 cd 进入该目录。
. 可选：使用这些校验和说明来确保您的二进制文件的真实性：
+
[source, powershell]
----
certutil -hashfile <downloaded_file>.zip SHA256
----
+
. 解压 .zip。您可以使用以下命令：
+
[source, powershell]
----
Expand-Archive -Path .\<downloaded_file>.zip -DestinationPath .
----
+
. 将此版本的 Java 添加到您的 `PATH`：
+
[source, powershell]
----
set PATH=%cd%\<extracted_directory>\bin;%PATH%
----
+
. 检查 Java 是否已正确安装：
+
[source, powershell]
----
java -version
----

== AIX

. 能够在 AIX 7.1 上运行的 Eclipse Temurin 的最新版本是 8u362、11.0.18 和 17.0.8。更高版本需要 IBM XL C/C++ 运行时包，可随编译器或独立提供，对于 JDK8 为 link:https://www.ibm.com/support/pages/ibm-xl-cc-runtime-aix-131[XLC 13.1]，对于 JDK11、17 和 21 为 link:https://www.ibm.com/support/pages/ibm-xl-cc-runtime-aix-161[XLC 16.1]，以及 link:https://www.ibm.com/docs/en/openxl-c-and-cpp-aix/17.1.2?topic=reference-open-xl-cc-runtime-environment-filesets[OpenXL 17]（这也需要 AIX 7.2 TL5 SP3 或更高版本）。
