= Instalación de Archivo
:page-authors: gdams, tellison, NickJavaDev88
:toc:
:icons: font

Los archivos proporcionan una forma sencilla de instalar Eclipse Temurin sin un
gestor de paquetes. Descargue el archivo adecuado desde la
link:/temurin/releases[página de versiones] y siga las instrucciones para su
plataforma a continuación.

[NOTE]
====
Para la mayoría de los usuarios, recomendamos usar un
link:/installation/linux[gestor de paquetes] o una
link:/installation/github-actions[integración CI/CD] en lugar de la
instalación manual por archivo. Los archivos son más adecuados para entornos
personalizados o plataformas sin soporte de gestor de paquetes.
====

== Verificación de su descarga

Todos los archivos de Eclipse Temurin incluyen un archivo de suma de
verificación SHA-256. Recomendamos encarecidamente verificar su descarga antes
de la instalación.

=== Linux / macOS / AIX / Solaris

[source, bash]
----
# Ejecute esto en el mismo directorio que el binario descargado
wget -O- -q -T 1 -t 1 <url_del_archivo_checksum> | sha256sum -c
----

=== Windows

[source, powershell]
----
certutil -hashfile <archivo_descargado>.zip SHA256
----

TIP: Los archivos de suma de verificación están disponibles junto a cada
binario en la link:/temurin/releases[página de versiones]. También puede
verificar las descargas a través de la
link:https://api.adoptium.net/q/swagger-ui/[API de Adoptium].

== Linux

[source, bash]
----
# Extraer el archivo
tar xzf <binario_openjdk>.tar.gz

# Agregar al PATH
export PATH=$PWD/<directorio_extraido>/bin:$PATH

# Verificar
java -version
----

Para hacer esto permanente, agregue la línea `export PATH` a su `~/.bashrc`,
`~/.zshrc`, o perfil de shell equivalente.

== macOS

[source, bash]
----
# Extraer el archivo
tar xzf <binario_openjdk>.tar.gz

# Agregar al PATH (note el subdirectorio Contents/Home)
export PATH=$PWD/<directorio_extraido>/Contents/Home/bin:$PATH

# Verificar
java -version
----

NOTE: Los archivos de macOS incluyen una estructura de directorios
`Contents/Home` que refleja el diseño estándar del JDK en macOS.

== Windows

[source, powershell]
----
# Extraer el archivo
Expand-Archive -Path .\<archivo_descargado>.zip -DestinationPath .

# Agregar al PATH (Símbolo del sistema)
set PATH=%cd%\<directorio_extraido>\bin;%PATH%

# Verificar
java -version
----

TIP: Para configurar `JAVA_HOME` y actualizar `PATH` de forma permanente en
Windows, use *Propiedades del Sistema > Variables de Entorno*, o ejecute:
`setx JAVA_HOME "C:\ruta\a\<directorio_extraido>"` y
`setx PATH "%JAVA_HOME%\bin;%PATH%"`.

== AIX

[WARNING]
====
Las últimas versiones de Eclipse Temurin capaces de ejecutarse en AIX 7.1
fueron 8u362, 11.0.18 y 17.0.8. Las versiones posteriores requieren el
paquete de runtime IBM XL C/C++:

* link:https://www.ibm.com/support/pages/ibm-xl-cc-runtime-aix-131[XLC 13.1] para JDK 8
* link:https://www.ibm.com/support/pages/ibm-xl-cc-runtime-aix-161[XLC 16.1] para JDK 11, 17 y 21
* link:https://www.ibm.com/docs/en/openxl-c-and-cpp-aix/17.1.2?topic=reference-open-xl-cc-runtime-environment-filesets[OpenXL 17] (requiere AIX 7.2 TL5 SP3 o posterior)
====

[source, bash]
----
# Extraer el archivo (tar de AIX no soporta la opción z)
gunzip -c <binario_openjdk>.tar.gz | tar xf -

# Agregar al PATH
export PATH=$PWD/<directorio_extraido>/bin:$PATH

# Verificar
java -version
----
